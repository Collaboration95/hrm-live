# Security policy

## Supported versions

Security fixes are considered for the latest release on the default branch.
Development and ad-hoc bundles are not public releases and should not be used
as trusted distribution artifacts.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability, exposed secret, or
unsafe release artifact. Once this repository is hosted on GitHub, use its
**Private vulnerability reporting** feature to send details to the repository
owner. Include the affected version or commit, reproduction steps, impact, and
any suggested mitigation.

If private reporting has not been enabled yet, contact the repository owner
privately through their GitHub profile rather than publishing exploit details.

The project aims to acknowledge reports within 7 business days and will
coordinate disclosure after a fix is available. Do not include BLE device
addresses, exported health data, signing certificates, notarization tokens, or
other secrets in reports.
